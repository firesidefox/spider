## 1. 目标

支持两类用户编辑场景：
- **Admin**：在用户管理页集中编辑任意用户（改角色、重置密码）
- **普通用户**：在导航栏账户区修改自己的密码（需验证旧密码）

## 2. 功能范围

### 2.1 Admin 编辑用户（UsersView）

- 表格每行增加"编辑"按钮，点击弹出编辑弹窗
- 弹窗字段：角色（select）、新密码（可选，留空不修改）
- Admin 不能修改自己的角色（已有后端限制，前端同步禁用）
- 提交调用已有 `PUT /api/v1/users/:id`

### 2.2 普通用户修改自己的密码

**新增后端端点：**
```
PUT /api/v1/me/password
Body: { old_password: string, new_password: string }
```
- 验证旧密码（bcrypt 比对）
- 新密码长度 ≥ 8
- 成功返回 `204 No Content`
- 旧密码错误返回 `403 { error: "invalid password" }`

**前端入口：**
- App.vue 导航栏 nav-user 区域，用户名旁加"改密码"按钮（或下拉）
- 点击弹出弹窗：旧密码、新密码、确认新密码三个字段
- 两次新密码不一致时前端直接提示，不发请求

## 3. 不在范围

- Admin 修改自己的密码走 2.2（普通用户流程），不走 2.1
- 用户名不可修改
- 忘记密码 / 邮件重置（无邮件系统）

## 4. 验收标准

- [ ] Admin 可在 UsersView 编辑任意其他用户的角色和密码
- [ ] Admin 编辑自己时角色字段禁用
- [ ] 任意登录用户可在导航栏修改自己的密码
- [ ] 旧密码错误时显示明确错误信息
- [ ] 两次新密码不一致时前端拦截
- [ ] 新密码 < 8 位时后端返回 400
- [ ] `go vet` 零错误，前端 `npm run build` 零错误
