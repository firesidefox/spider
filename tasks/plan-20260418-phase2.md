# 实现计划：Phase 2 — 用户管理与认证

**日期：** 2026-04-18  
**Spec：** docs/spec-20260418-phase2-user-management.md

---

## 概述

为 spider 添加多用户认证体系：JWT（Web UI）+ API Token（MCP），三级 RBAC，用户管理界面。认证默认关闭（`auth.enabled: false`），显式开启后生效。

---

## 架构决策

1. **无新 Service 层**：用户/Token 逻辑简单，handler 直接调用 store，避免过度抽象。
2. **中间件注入 UserContext**：认证结果注入 `context.Context`，handler 通过 `ctx.Value` 取用，不改变函数签名。
3. **auth.enabled 开关**：Config 新增 `Auth.Enabled bool`，false 时中间件直接放行，现有功能零影响。
4. **JWT 黑名单内存存储**：logout 写内存 map，重启清空，可接受（24h TTL）。
5. **前端路由守卫**：`useAuth` composable 管理登录态，router beforeEach 拦截未登录请求。

## 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| DB migration 破坏现有数据 | 高 | ALTER TABLE 幂等，新列可空，测试前备份 |
| JWT 密钥丢失（重启） | 中 | 持久化到 `jwt.key` 文件 |
| auth.enabled 开关遗漏某路由 | 中 | 中间件统一包裹，不逐路由处理 |
| 前端 fetch 未带 token | 低 | useAuth 统一封装 fetch，不散落各处 |
